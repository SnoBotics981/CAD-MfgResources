All of the testing notes/procedure/plans shall either live on this wiki page or be referenced from here.

  • Basic Philosophy
    • The IVP parameters provided by Steve Collins are intended to result in better pointing accuracy of the HGA (High Gain Antenna). Steve has provided the following:
      • Updated parameters,
      • Matlab code and plots showing the pointing error with the new and old IVP values,
      • CSV files of the pointing errors,
      • ...
    • From the data Steve provided I created .xlsx files which seemed easier to manipulate. I also created a tar ball of the data from Steve and that tar ball is (will be) attached to this wiki.
    • In an effort to fly as we test and test as we fly the FLTS machines used for this testing will be configured to boot into RAM mode. (This change has NOT been made yet.)
  • Software Changes Included With This Test
    • Change for handling smaller select_surface files is included in the build
    • End of Epoch change is included
    • Others ...
  • Build locations and descriptions.
    • There are two builds that will be used, one with the new IVP parameters and one using the old. Both of these version are built using the R9.5 version of the software, but with different compiler options. Check with the FSW lead for the most current sandbox. Currently these are 
      /proj/mer-fswdev/steele/merfsw-pkg-R9-5-20160517-testHgaNewIvp
      and 
      /proj/mer-fswdev/steele/merfsw-pkg-R9-5-20160517-testHgaOldIvp
    • A third build will be built that will contain the new IVP constants, but will not have the test code inserted. This code should behave exactly like the the 'testHgaNewIvp' version, but will not create any output test files. Rather the analsys will involved using the EHA data directly to verify the same results as obtained from the other 'testHgaNewIvp' build. The EHA data is only output every 5 minutes, the the output files created from the 'testHgaNewIvp' build are output every 1 second.

  • Testing Notes
    • The basic idea is that tests will be run from the same date and time on both builds. The output from each run will be analyzed for the pointing position and accuracy of the HGA. The pointing accuracy of the software version that is using the new IVP parameters should have pointing errors of less than 0.5 degrees and should be approximately 10-20 times more accurate than the runs made with the software that is using the R9.4 IVP parameters.
    • AVSIM should be started and the system run from Surface Nominal mode and the HGA initialized. The basic commands for a run are:
      • Start AVSIM: (Note that the default from the start_avsim-rds is the R9.5 new IVP version of the FSW.)
        • /home/steele/scripts/start_avsim-rds -f07 --surface_nominal --r=R9.5newIVP --no_dmd
      • After getting into surface nominal execute the commands to jump the clock as desired.
        • CMD_CNSTRNT_OVRD
        • SET_SC_CLOCK 2014-109T01:00:00
      • Then execute the command for the HGA to track the earth, using 1000 seconds as the 'nominal' time period
        • HGA_TRACK_EARTH 1000,AUTO
      • Then copy the data from mermoose (or slammer) to a machine that can run matlab to analyze the data.

        packet_report -eha fsts07_t8_1d4_mer1_1.16.172.152135.pkt
